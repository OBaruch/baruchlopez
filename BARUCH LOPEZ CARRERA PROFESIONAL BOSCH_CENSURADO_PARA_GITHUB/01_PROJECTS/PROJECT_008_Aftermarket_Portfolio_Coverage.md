# Análisis de Cobertura de Portafolio (Aftermarket Automotriz)

> Contenido anonimizado.

## Resumen
Análisis de **cobertura y brechas (gap)** del portafolio de un producto de **aftermarket automotriz** para Norteamérica, ponderado por **parque vehicular y ventas**, implementado con **KNIME** (incluye ML) y **Power BI**.

## Mi participación
Participación **acotada**: **actualicé los nodos de un workflow de KNIME** para resolver una **discrepancia entre los outputs esperados por Power BI y los generados por KNIME**, alineando los formatos de salida. *(El análisis de cobertura completo fue desarrollo de equipo.)*

## Tecnologías usadas
- **KNIME** (workflows de datos; nodos de ML como árboles de decisión / reglas de asociación / PMML; Python embebido).
- **Power BI** (modelo y dashboard de aplicaciones).
- Datos de **parque vehicular** y ventas; catálogos de aplicaciones.

## Actividades realizadas
- Ajuste de nodos de KNIME y de los archivos de salida para igualar el formato esperado por Power BI.

## Impacto
- Alineación de outputs KNIME ↔ Power BI para visibilidad de cobertura/gap.

## Complejidad técnica
Media: depuración de un workflow KNIME con ML y armonización de formatos de datos.
